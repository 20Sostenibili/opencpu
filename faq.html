Frequently Asked Questions

What is Open Computing?
Open Computing is a term coined by Jeroen Ooms in his (ongoing) PhD research at UCLA Department of Statistics. It describes a philosophy about embedded, scalable, transparent, reproducible scientific computing. OpenCPU is the name of the software based on R that implements some of these ideas. If you are interested in reading more about the motivations and philosophy of open computing, have a look this early draft of the research proposal.
What is OpenCPU?
OpenCPU is a web development framework that implements some of the principles of Open Computing. Here are some statements about OpenCPU:
R Web Development
OpenCPU is the name of a web development framework for scientific computing based on R.
HTTP REST API
OpenCPU exposes a REST API to R. Client applications can read, write and execute R code remotely through the API
Abstraction layer
OpenCPU is an abstraction layer that provides a convenient interface to R functionality while hiding some of the technicalities that come with R and Statistical computing.
Remove Function Calls
OpenCPU defines a direct mapping between HTTP requests and R function calls.
Open Source R-X
Everything on OpenCPU is by definition Open Source. A helpful UNIX analogy: you have both Read and Execute access on all code that lives on the server. (And write access to your home dir)
Custom code
Conventional web services expose a limited number of parameterized procedures. Arbitrary code execution is explicitly prevented. However, in OpenCPU clients can run, share, execute custom code.
Apps
OpenCPU allows you to easily create and publish R web applications.
What can I use OpenCPU for?
You can use OpenCPU to power application with live statistical computations in the cloud. Any functionality available in R can be called remotely and embedded into systems or web applications.
For example you can use OpenCPU to do live forecasting of timeseries data and calculate predictive analytics. Use smoothing, correlations, (logistic) regression. Or you just need something to create a pretty plot of your data.
Where do I begin?
To use OpenCPU you need to have some familiarity with R and Web Development. If you have never heard about R have a look at one of the "Introduction to R" books/articles or read one of the many R blogs. After having a look at the documentation page, a good place to start would be to implement a simple web application based on jQuery.
What is R? Do I need to know R?
R is an open source statistical computing language. It is widely popular among scientists, researchers and analysis. The OpenCPU API interfaces to R. Some understanding of R is needed to use OpenCPU. Fortunately, there have been many introduction books, articles and blogs written about R.
Can I install my own OpenCPU server?
Absolutely. Have a look at the downloads page and license page.
What about security?
Security is maybe the most challenging aspect of open computing. We want users to be free to run any code, but at the same time prevent malicious use of public resources. Currently OpenCPU relies on several security technologies, the most important one is AppArmor: a security module for the Linux kernel that enforces mandatory access control. This seem to work reasonably well, although the line between regular and malicous use can be very thin. If you are finding vulnerabilities or have suggestions for improvements, please do let us know.
Does OpenCPU support { Windows | OSX | RHEL | etc } ?

You are asking the wrong question. OpenCPU is a cloud computing service. It is designed to run on its own server in the cloud or on some virtual machine. Once installed, you will be interacting with OpenCPU mostly through the API. It not recommended to host other software on the same server as OpenCPU, as R is very greedy with resources, and it is also unwise from a security point of view. Instead, just spin up a little virtual server for OpenCPU using VMWare, EC2 or whatever you are comfortable with.

You can develop your clients or software stack in any language, on any operating system, as long as it speaks HTTP. The actual OS of the OpenCPU server is not more relevant than the OS on your printer or vpn server: you will rarely directly interact with it. That said, OpenCPU runs Ubuntu Linux. If you really want to port it on another OS, you're on your own. Especially the sandboxing can be tricky on other operating systems. But of course we encourage anyone to try.

What are OpenCPU apps?
An OpenCPU App is an R package that also contains a web application. By putting your web application inside the /inst/opencpu/apps directory of your R package, it will automatically be deployed when the R package is installed on an OpenCPU server. This way you can ship both the front-end and backend of your R web application in one simple deployable package.
Is OpenCPU compatible with [ Software package X ] ?
As long as your client speaks HTTP you should be able to call OpenCPU API. It is highly recommended to install OpenCPU on its own server. That way, it cannot conflict with anything else on the server.
What is Cloud Computing?
Cloud Computing is a buzz word that refers to the general practice of running software on centralized, shared servers as opposed to your local machine. This makes software easier to maintain and we can make more efficient use of resources.
What is Open Source software?
Open Source Software, or Free Software is software for which the author has release the software under a license that allow to copy, distribute or modify the software. It is the opposite of proprietary software, where the author generally charges fees for using the software, and copying or modifying is not allowed.
Can OpenCPU be used to build proprietary applications?
Absolutely. Have a look at the license page for the conditions.
Is OpenCPU stricktly REST?
We try to respect the REST principles where possible, but some of the APIs are maybe more RPC style than REST.
